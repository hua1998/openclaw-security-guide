# 研究报告格式规范

> 兼容所有交互方式的统一报告格式

---

## 背景

不同交互方式效果差异：
- IM 机器人：文字为主，格式有限
- Web 会话：支持富媒体
- Agent 对话：结构化输出

---

## 统一报告格式

### 核心原则

1. **纯文本优先** - 所有渠道都能显示
2. **Markdown 兼容** - 标准格式
3. **降级方案** - 自动适配

---

## 格式结构

### 1. 标题行

```
# 报告标题
```

### 2. 一句话摘要

```
> 一句话说明核心结论
```

### 3. 关键指标 (表格)

```
| 指标 | 值 |
|------|-----|
| 安全评分 | 85 |
| 风险数 | 3 |
```

### 4. 风险清单 (列表)

```
### 风险
- P0: 高风险1
- P1: 中风险1
```

### 5. 建议 (列表)

```
### 建议
1. 建议1
2. 建议2
```

### 6. 详情链接

```
详见: https://...
```

---

## 格式适配

### IM/机器人

- 纯文本
- Emoji 符号
- 链接降级

### Web/文档

- 完整 Markdown
- 富媒体支持
- 表格图表

### Agent

- 结构化 JSON
- 可解析
- 自动处理

---

## 示例

### 输入 (Agent)

```json
{
  "score": 85,
  "risks": ["P0: 配置泄漏"],
  "suggestions": ["关闭调试"]
}
```

### 输出 (统一)

```
# 安全评估报告

> 整体安全评分 85/100

## 风险
- P0: 配置泄漏
- P1: 权限过大

## 建议
1. 关闭调试模式
2. 收窄权限

详见: https://gitee.com/...
```

---

## 交付约定

| 场景 | 格式 | 说明 |
|------|------|------|
| IM 推送 | 纯文本+Emoji | 快速知悉 |
| 邮件 | Markdown | 详细阅读 |
| Web | HTML | 完整展示 |
| Agent | JSON | 程序处理 |
