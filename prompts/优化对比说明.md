# OpenClaw安全评估提示词优化对比

## 优化前后对比

### 原提示词 V1
```
我需要你作为OpenClaw安全治理专家。
数据源：https://github.com/hua1998/openclaw-security-guide 
https://gitee.com/caidongyun/agent-security-guide
请先读取，然后对我的AI Agent进行安全评估，输出：
1. 安全评分 (0-100)
2. 风险清单 (P0/P1/P2)
3. 加固建议
```

### 优化后 V2
见 [security-assessment-v2.md](security-assessment-v2.md)

## 具体改进详情

### 1. 评分体系细化

**V1**: 只有一个总分
```
安全评分: 23/100
```

**V2**: 多维度加权
```
| 安全维度 | 得分 | 权重 |
| 认证与访问控制 | 15/100 | 20% |
| 数据与隐私保护 | 30/100 | 15% |
| 网络安全 | 20/100 | 15% |
| 应用安全 | 25/100 | 15% |
| 运行时安全 | 10/100 | 15% |
| 审计与合规 | 40/100 | 20% |
| 综合评分 | 23/100 | - |
| 评级 | D (不及格) | - |
```

### 2. 风险分级明确化

**V1**: 仅有优先级标识
```
风险清单 (P0/P1/P2)
```

**V2**: 明确时间要求 + 修复命令
```
🔴 P0 - 立即修复 (0-24小时)
🟠 P1 - 短期修复 (1-7天)
🟡 P2 - 计划修复 (1-4周)
🟢 P3 - 持续改进
```

### 3. 加固建议可执行

**V1**: 描述性建议
```
加固建议: 启用Token认证
```

**V2**: 可执行配置
```bash
# 立即执行 - 一键加固脚本
#!/bin/bash
# 生成32位安全Token
TOKEN=$(openssl rand -hex 16)
# 更新配置
jq '.gateway.auth.mode = "token" | .gateway.auth.token = "'$TOKEN'"' config.json
```

### 4. 新增合规映射

**V2新增**:
```
| 检测项 | 等保2.0 | OWASP Top 10 | NIST CSF |
| 认证机制 | 8.1.4 | A07 | PR.AC-1 |
| 数据加密 | 8.1.5 | A02 | PR.DS-1 |
```

### 5. 复测验证机制

**V2新增**:
```bash
# 修复后验证
python tools/security_detector.py --config config.json --json
# 预期: 评分 >= 90, 无P0/P1风险
```

## 推荐使用方法

### 方法1: 直接复制优化版
复制 [security-assessment-v2.md](security-assessment-v2.md) 内容到ChatGPT/Claude使用

### 方法2: 分阶段输入
```
第一步: 发送"角色定义"部分建立上下文
第二步: 发送"评估标准"部分明确范围
第三步: 发送"输出格式"部分规范输出
第四步: 提供具体配置开始评估
```

### 方法3: 保存为系统提示词
在支持自定义助手的平台，将V2内容保存为系统提示词
